# 房間

> 優先級：**P0 MVP**
> 相關 flow：[docs/flows/02-login-to-game.md](../../flows/02-login-to-game.md), [docs/flows/03-room-lifecycle.md](../../flows/03-room-lifecycle.md)
> 相關 system：[docs/systems/room-matchmaking.md](../../systems/room-matchmaking.md)

## 1. 目的

玩家在等待開局前的準備空間：看隊友、選曲、按準備、房主開始遊戲。

- **進入條件**：從大廳加入或建立房間
- **成功離開（開始）**：全員準備 + 房主按開始 → 跳轉 `05-game-arena`
- **成功離開（退出）**：按離開 → 回 `03-lobby`

## 2. 原版行為（Reference）

- 顯示房間內所有玩家（名字、準備狀態、角色外觀）
- 房主有特殊標記（皇冠等）
- 玩家可切換「準備 / 未準備」
- 房主可選曲（歌曲列表）
- 房主可選模式（普通/自由/反鍵等）
- 房主按「開始」→ 全員進入遊戲場地
- 房主可踢人
- 玩家可按「離開」回大廳
- 可能有房間聊天

> 待考據。見 [flows/03-room-lifecycle.md](../../flows/03-room-lifecycle.md)

## 3. 保留（Keep）

- 顯示玩家列表與準備狀態
- 房主選曲
- 全員準備才能開始
- 房主按開始
- 離開回大廳
- 區分房主與一般玩家權限

## 4. 改版 / 優化（Change）

| 項目 | MVP | 之後 | 原因 |
|------|-----|------|------|
| 模式選擇 | **自由 / 普通** + **向上/向下/傾斜** | 房主選模式+方向；普通鎖難度 | MVP P0 先自由+向上、普通+向上 |
| 歌曲數量 | 3～5 首測試曲 | 擴充曲庫 | osu import |
| 踢人 | 做 | — | 基本管理 |
| 房間密碼 | 不做 | P2 | 簡化 |
| 角色 3D 展示 | 2D 頭像列表 | P2 | 降低成本 |
| 房間聊天 | 不做 | P1 | 非核心 |
| 換位置 | 不做 | P2 | 簡化 |
| 房主轉移 | 簡化為解散 | P1 | MVP 降低複雜度 |

## 6. 資料與狀態

| 時機 | 資料 |
|------|------|
| 進入前 | userId, serverId |
| 在房間內 | roomId, isHost, seatIndex, isReady, songId, mode |
| 離開時（開始） | + gameSessionId |
| 離開時（退出） | 清除 roomId |

## 7. 邊界與錯誤

| 情況 | 處理 |
|------|------|
| 有人未準備就按開始 | 擋下，提示「尚有玩家未準備」 |
| 房主離開 | MVP：解散房間，其他人回大廳 |
| 被踢 | 顯示提示，回大廳 |
| 倒數時有人離開 | 待確認：取消開始 or 繼續 |
| 選曲後有人換曲 | 重置所有人準備狀態（待確認） |
