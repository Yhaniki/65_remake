# 大廳

> 優先級：**P0 MVP**
> 相關 flow：[docs/flows/02-login-to-game.md](../../flows/02-login-to-game.md)
> 相關 system：[docs/systems/room-matchmaking.md](../../systems/room-matchmaking.md), [docs/systems/chat.md](../../systems/chat.md)

## 1. 目的

玩家瀏覽公開房間列表，建立或加入房間，是大廳社交的核心 hub。

- **進入條件**：選服成功
- **成功離開**：加入/建立房間 → 跳轉 `04-room`

## 2. 原版行為（Reference）

- 顯示當前頻道/大廳的房間列表
- 每個房間顯示：房號、房主名、人數、模式、歌曲（若在玩）
- 可建立新房間
- 可點房加入
- 可能有頻道切換
- 可能有聊天視窗
- 入口按鈕（post-MVP）：商場、家族、個人房

> 待考據原版 UI 布局。見 [original-overview.md](../../reference/original-overview.md)

## 3. 保留（Keep）

- 房間列表顯示
- 建立房間
- 加入房間
- 顯示房間基本資訊（房主、人數、狀態）

## 4. 改版 / 優化（Change）

| 項目 | MVP | 之後 | 原因 |
|------|-----|------|------|
| 頻道切換 | 不做（單頻道） | P1 | 簡化 |
| 聊天 | 不做或極簡 | P1 | 非核心 |
| 快速加入 | 可選 | P1 | 方便配對 |
| 房間篩選/搜尋 | 不做 | P2 | MVP 房少 |
| 商場/家族/個人房入口 | 隱藏或灰掉 | post-MVP | 見 06/07/08 |
| 3D 大廳場景 | 2D 列表即可 | P2 | 降低成本 |
| 玩家虛擬形象走動 | 不做 | post-MVP | 非 MVP |

## 6. 資料與狀態

| 時機 | 資料 |
|------|------|
| 進入前 | userId, sessionToken, serverId |
| 離開時 | + roomId（若加入房間） |

## 7. 邊界與錯誤

| 情況 | 處理 |
|------|------|
| 房間已滿 | 提示「房間已滿」，留在大廳 |
| 房間已開始遊戲 | 不可加入，提示 |
| 房間不存在（已被刪） | 刷新列表，提示 |
| 建立房間失敗 | 提示錯誤 |
| 斷線 | 回登入或重連提示 |
