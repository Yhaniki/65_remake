# 登入介面

> 優先級：**P0 MVP**
> 相關 flow：[docs/flows/02-login-to-game.md](../../flows/02-login-to-game.md)
> 相關 system：[docs/systems/account-auth.md](../../systems/account-auth.md)

## 1. 目的

玩家輸入帳號密碼，驗證身份後進入伺服器選擇。

- **進入條件**：啟動遊戲
- **成功離開**：登入驗證通過 → 跳轉 `02-server-select`

## 2. 原版行為（Reference）

- 帳號、密碼輸入框
- 登入按鈕
- 可能有的元素（待考據）：
  - 記住帳號
  - 註冊連結
  - 忘記密碼
  - 版本號顯示
- 登入失敗顯示錯誤訊息
- 登入成功 → 伺服器選擇畫面

> 細節待原版截圖補充。見 [docs/reference/original-overview.md](../../reference/original-overview.md)

## 3. 保留（Keep）

- 帳號 + 密碼登入
- 登入失敗有明確錯誤提示
- 登入成功跳轉選服

## 4. 改版 / 優化（Change）

| 項目 | MVP | 之後 | 原因 |
|------|-----|------|------|
| 訪客登入 | 不做 | P1 考慮 | 簡化帳號系統 |
| 記住帳號 | 可選做 | — | 提升回訪體驗 |
| 註冊流程 | 簡化或另頁 | — | MVP 可預建測試帳號 |
| UI 風格 | 現代簡潔 | — | 不必復刻 2007 UI |
| 背景動畫 | 靜態或簡單動效 | P1 | 降低製作成本 |

## 6. 資料與狀態

| 時機 | 資料 |
|------|------|
| 進入前 | 無（或上次記住的帳號名） |
| 離開時 | userId, sessionToken, displayName |

## 7. 邊界與錯誤

| 情況 | 處理 |
|------|------|
| 帳密錯誤 | 顯示「帳號或密碼錯誤」，留在此頁 |
| 帳號不存在 | 同上或分開提示（待確認） |
| 伺服器連不上 | 顯示「無法連線，請稍後再試」 |
| 已在其他地方登入 | 待確認原版行為 |
