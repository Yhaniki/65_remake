# UI 規範（UI Patterns）

> 全站共用的 UI 元件與交互模式。具體 screen 布局見各 spec.md。

## 設計原則

1. **簡潔優先** — MVP 用 2D 界面，不追求 3D 大廳
2. **一致性** — 同類操作同位置（返回左上、確認右下）
3. **反馈明確** — 每個操作都有視覺/文字反馈
4. **可讀性** — 繁體中文，字體適中

## 共用元件

### 按鈕

| 類型 | 用途 | 樣式 |
|------|------|------|
| Primary | 主要操作（登入、開始、確認） | 醒目色，大尺寸 |
| Secondary | 次要操作（返回、取消） | 灰色或描邊 |
| Danger | 危險操作（登出、踢人） | 紅色或警告色 |
| Disabled | 不可操作 | 灰色，不可點 |

### 輸入框

- 帳號、密碼等文字輸入
- 密碼預設隱藏（••••）
- 錯誤時紅框 + 下方錯誤文字

### 列表

- 房間列表、玩家列表、伺服器列表
- 每行可點擊
- 選中態：高亮背景
- 空列表：顯示「目前沒有房間」等提示

### 狀態標記

| 標記 | 意思 |
|------|------|
| ★ | 房主 |
| ✓ | 已準備 |
| ● | 在線 |
| ○ | 離線/空位 |

### 提示 / Toast

- 成功：綠色，3 秒消失
- 錯誤：紅色，需手動關或 5 秒消失
- 位置：畫面中上方

### 確認對話框

- 危險操作前弹出：「確定要離開房間嗎？」
- [取消] [確定]

## 布局慣例

```
┌─────────────────────────────────┐
│  [← 返回]    標題    [操作按鈕]  │  ← 頂部導航列
├─────────────────────────────────┤
│                                 │
│           主要內容區              │
│                                 │
├─────────────────────────────────┤
│         [ 主操作按鈕 ]            │  ← 底部操作列（可選）
└─────────────────────────────────┘
```

## 字體

- 中文：待確認（建议 Noto Sans TC 或微软正黑）
- 英文/數字：同上或搭配 Inter
- 分數/Combo：等寬或特殊數字字體

## 響應式

- MVP 鎖定 **16:9** 視窗（如 1280×720 或 1920×1080）
- 不做手機適配
