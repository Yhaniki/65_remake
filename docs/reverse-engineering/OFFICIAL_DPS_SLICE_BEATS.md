# 官方 DPS 切片拍數統計

## 結論

- 官方 DPS 的 dance row 最常見是 **8 拍切一次**。
- 在可解析的標準 `PAS00003` DPS 中，8 拍 row 有 **15,609 / 51,694** 筆，占 **30.19%**。
- 若以「每首 DPS 最常出現的 row 拍數」來看，8 拍也是第一名：**642 / 1,619** 首，占 **39.65%**。
- 主要分布不是固定單一值，而是集中在 **6-10 拍**：dance row 約 **76.22%**；放寬到 **5-11 拍**約 **88.46%**。
- 若把 row 內的 sub-item 依 SongList timing 換算成實際段落拍數，中央値約 **7.04 拍**，平均 **7.057 拍**，10-90 百分位約 **2.971..11.01 拍**。

實作建議：random/compose 產生官方風格 DPS 時，預設用 **8 拍**是最貼近官方的選擇；若要更自然，可在 **6-10 拍**或 **5-11 拍**間依 MOT 長度和節奏微調。

## SongList Timing 欄位

本次沒有全量解 K.gn。只抽樣校準欄位後，全量統計使用 SongList raw record：

| 資料 | SongList 欄位 / offset | 用途 |
|---|---:|---|
| extra 值 | `max_m_e/n/h`，record offset `508/512/516` | 計算 effBPM |
| duration 秒 | record offset `728/732/736` (`uint32`) | 計算 effBPM |
| effBPM | `max(extra) * 4 * 60 / max(duration)` | 將 `v252` 秒數換回拍數 |

註：`songlist_archive.py` 目前把 offset `508/512/516` 命名為 `max_m_e/n/h`，但抽樣 K.gn 對照顯示它對應 GN timing 用的 extra 值。

## 判讀

- 如果目標是「row 邊界多久切一次」，答案是 **8 拍最典型**。
- 如果目標是「每段 MOT 動作多久換一次」，官方常因 `sub_count=2` 在同 row 內塞多段，所以實際 sub-item 段落更分散，約 **5-10 拍**最常見，中位數約 **7 拍**。
- `pre_b` 超過 16 拍的少數資料多半是特殊段、尾段、長 idle 或特殊 DPS，不適合作為 random DPS 的常規切片長度。
- 產生器若要像官方：建議 row `pre_b` 以 **8 拍**為中心，允許 **6-10 拍**；若要減少 row 邊界造成的 idle/fade，可使用 `sub_count=2` 或 compose 內合段策略。
